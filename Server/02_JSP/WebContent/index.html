JSP (Java Server Page)

: HTML 형식에서 자바 코드를 사용할 수 있는 "자바 언어"로
서블릿에서 Java 코드로 응답 HTML 을 구현했던 복잡함을 간단하게 작성, 해결 가능

1. JSP 스크립트 요소 (Elements)

: JSP 페이지에 자바 코드를 직접 기술할 수 있게 해주는 기능

  1. 선언문 (<%! 자바 코드%>) : 멤버 변수와 메소드를 선언하기 위해 사용
  2. 스크립트릿 (<% 자바 코드%>) : JSP 에서 자바 코드를 사용하기 위해 사용
  3. 표현식 (<%= 자바 코드%>) : 자바에서 작성한 값을 출력할 때 사용

2. 지시어 (Directive)

: JSP 페이지 전체에 영향을 미치는 정보를 기술할 때 사용

  1. page : 현재 JSP 페이지를 컨테이너에서 처리하는 데 필요한 각종 속성 (보통 맨 앞에 위치)
    • language : JSP 페이지 내부에서 사용할 스크립트 언어 지정
    • contentType : 웹 브라우저가 받아 볼 페이지의 형식, 인코딩 방식 지정
    • pageEncoding : JSP 파일에 기록된 자바 코드의 인코딩 방식 지정
    • import : 자바의 import 와 같은 의미로, JSP 페이지에서 사용할 클래스를 import 하는데 사용됨
    • errorPage : JSP 페이지에서 예외가 발생했을 때 해당 예외를 처리할 페이지 (HTML, JSP) 지정
    • isError : 현재 페이지가 다른 페이지에서 발생한 예외를 처리하는 페이지라는 걸 지정
    • 등등.....
  2. include : JSP 페이지에서 다른 JSP 파일 또는 HTML 파일을 삽입할 수 있도록 하는 지시어
  3. taglib : JSP 기능을 확장할 수 있는 사용자 정의 태그를 가져와서 사용할 수 있는 지시어

내장 객체

: JSP가 자바 코드로 자동으로 생성될 떄, 자바 파일에 자동으로 추가되는 객체들
즉, JSP 페이지 내부에서 개발자가 직접 선언하지 않아도 바로 사용이 가능한 객체

  1. request : 웹 브라우저의 요청 정보를 저장하고 있는 객체
  2. response : 웹 브라우저의 요청에 대한 응답을 나타내는 객체
  3. session : 웹 브라우저의 정보를 유지하기 위한 세션 정보를 저장하고 있는 객체
  4. out : 화면에 출력을 위한 출력 스트림 객체
  5. application : 웹 어플리케이션(컨텍스트)의 실행 환경을 제공하는 서버의 정보와 서버 측 자원에 대한 정보를 저장하고 있는 객체
  6. config : JSP 페이지에 대한 설정 정보를 저장하고 있는 객체
  7. page : JSP 페이지를 구현한 서블릿 인스턴스 자체를 참조하는 객체 (page = this;)
  8. exception : JSP 페이지에서 예외가 발생하였을 경우 에러 페이지에 전달되는 예외 객체
  9. pageContext : JSP 페이지와 관련된 다른 내장 객체를 얻어내거나 현재 페이지의 요청과 응답의 제어권을 다른 페이지로 넘겨주는데 사용되는 객체

영역 객체

: JSP 에서 제공하는 내장 객체들 중 pageContext, request, session, application 객체들은 해당 객체에 정의된 유효 범위를 가짐
해당 객체들은 유효한 범위 안에서 데이터를 서로 공유할 수 있음
이 공유되는 데이터를 속성(Attribute)라고 하며,
속성을 공유할 수 있는 유효 범위를 영역(Scope) 라고 함

  1. page : pageContext 내장 객체를 통해 접근할 수 있는 영역으로, 해당 페이지가 웹 브라우저에 서비스를 제공하는 동안에만 유효
  2. request : request 내장 객체를 통해 접근할 수 있는 영역으로, 웹 브라우저의 요청이 처리되는 동안에만 유효
  3. session : session 내장 객체를 통해 접근할 수 있는 영역으로, 세션이 유지되고 있는 범위 안에서 유효
  4. application : application 내장 객체를 통해 접근할 수 있는 영역으로, 웹 애플리케이션이 실행되고 있는 동안에만 유효
